# MCP OAuth Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为每个登录用户提供可配置的远程Streamable HTTP MCP连接，内置Notion OAuth，并让Agent在运行流图中展示MCP调用及写操作审批。

**Architecture:** 后端使用稳定版官方MCP Python SDK连接远程服务器，`McpConfigService`持久化用户级连接和加密凭据，`McpOAuthCoordinator`处理标准OAuth发现、PKCE、动态注册及刷新，`McpRunSessionPool`把远程工具适配到现有ToolRegistry。写入、删除和风险未知调用通过进程内`ApprovalBroker`暂停Agent线程，由独立认证API唤醒；现有SSE继续传输`agent_step`，并增加审批事件。前端在当前“工具与MCP”页面管理服务器，在聊天消息和运行抽屉中复用同一个审批组件。

**Tech Stack:** FastAPI、SQLAlchemy、Pydantic v2、官方`mcp==1.28.1`稳定版SDK、httpx、jsonschema、SQLite/MySQL、React 18、Vite、项目现有`tests/check_*.py`契约测试。

- 不修改或删除现有`tool_config`、`agent_tool_call`和`chat_message.trace_json`数据。

```python
def execute_rowcount(
    sql: str,
    params: dict[str, Any] | None = None,
) -> int:
    with db.engine.begin() as conn:
        result = conn.execute(text(sql), params or {})
        return int(result.rowcount or 0)
```

- [ ] **Step 4: 实现`McpConfigService`**

服务构造器接收`fetch_one`、`fetch_all`、`execute`、`execute_rowcount`、`cipher`和`now_str`。凭据使用：

```python
def encrypt_credentials(self, value: dict[str, Any]) -> str:
    return self.cipher.encrypt(
        json.dumps(value, ensure_ascii=False, separators=(",", ":"))
    )

def decrypt_credentials(self, value: str | None) -> dict[str, Any]:
    raw = self.cipher.decrypt(value)
    if not raw:
        return {}
    parsed = json.loads(raw)
    return parsed if isinstance(parsed, dict) else {}
```

实现`list_for_user`、`get_owned`、`create_server`、`update_server`、`delete_server`、`save_credentials`、`secret`、`save_tool_snapshot`、`set_status`、`create_oauth_session`、`consume_oauth_session`和`delete_expired_oauth_sessions`。所有SQL必须包含`user_id`；OAuth消费先读取，再用`execute_rowcount`删除同一`id + user_id + state_hash`，只有删除1行才返回会话。

`save_tool_snapshot()`使用以下首次启用规则：

- 工具数不超过`MCP_MAX_EXPOSED_TOOLS`且没有旧快照时，首次连接默认启用全部工具。
- 工具数超过限制时，首次连接保存空的`enabled_tools_json`，要求用户明确选择。
- 刷新已有连接时，只保留仍然存在的已启用工具，不自动启用新出现的工具。
- 发现失败时不调用`save_tool_snapshot()`，因此旧快照保持不变。

- [ ] **Step 4: 实现安全结果归一化**

工具结果规则：

```python
return {
    "structuredContent": structured
    if isinstance(structured, (dict, list))
    else None,
    "content": text[:max_chars],
    "isError": bool(result.isError),
}
```

非文本内容只记录公开类型，不序列化二进制。序列化后的UTF-8字节数超过`MCP_MAX_RESPONSE_BYTES`立即失败。

工具发现同样限制总响应大小；单个工具说明截断到1000字符，输入schema必须是JSON对象。无效或超限的目录不能覆盖上一次成功快照。
