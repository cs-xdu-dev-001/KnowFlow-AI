# KnowFlow AI Frontend Professional Redesign

## Goal

把现有前端从“功能堆叠式 demo”改成一个专业 AI 工作台。页面要学习 Codex、Claude、ChatGPT、Cherry Studio 这类产品的交互范式，但不照搬任何开源项目代码或品牌视觉。

## Product Direction

第一版聚焦三个核心场景：对话、知识库、模型设置。Agent 与 Notion 暂时不作为主链路展示，避免产品看起来贪多。RAG 不再作为手动按钮，而是由“是否选择知识库”决定：不选择知识库就是普通对话，选择知识库就自动检索并展示引用依据。

## Layout

整体采用三段式专业工作台：

- 左侧窄侧边栏：品牌、可折叠菜单、新建对话、最近会话、知识库快捷入口、运行状态。
- 中央主工作区：对话页为主，包含模型选择、知识库上下文、消息流、建议提示、输入框。
- 右侧证据抽屉：默认可见但可收起，用于展示引用片段和检索过程，避免把调试信息塞进主对话。

知识库页改成文档工作区：左侧是知识库列表和创建入口，右侧是文档入库、文档列表、切片预览和检索测试。设置页改成供应商与模型配置中心：供应商选择、常用模型预设、Base URL 自动填充、API 文档入口。

## Visual System

视觉走浅色、克制、高密度的 AI 工具风：

- 背景使用低饱和灰白，不使用大面积蓝紫渐变。
- 主按钮只保留在高优先级操作，例如发送、创建、保存。
- 卡片半径控制在 8px 到 12px，减少“花哨面板感”。
- 字体层级收紧，页面标题、工具栏、表单标签和消息体有明确大小差异。
- 使用文本图标与少量符号，不引入外部图标依赖，保证 FastAPI 静态托管可直接运行。

## Interactions

关键交互包括：

- 侧边栏折叠后保留图标入口。
- 证据面板可收起，适合专注聊天。
- 输入框支持 Enter 发送、Shift+Enter 换行。
- 选择供应商时自动填充 Base URL 与默认模型。
- 常用模型下拉覆盖 OpenAI、DeepSeek、MiMo、百炼、Gemini、MiniMax 等。
- 对话页只有一个“发送”入口，系统根据知识库选择自动决定是否启用 RAG。
- 引用片段、检索过程、复制回答、空状态、加载状态都要有明确反馈。

## Implementation Boundary

为避免引入构建链路，前端继续使用 FastAPI 静态托管的 `index.html`、`styles.css`、`app.js`，但按组件化思路重写结构和交互。后端接口保持不变，不修改核心业务逻辑。

## Verification

新增静态前端结构测试，检查以下内容：

- 页面包含专业工作台结构：侧边栏、聊天主区、证据抽屉、知识库页、设置页。
- 文案为中文且没有常见乱码片段。
- 模型供应商包含 MiMo，并支持供应商预设。
- 聊天只有统一发送入口，不再暴露 Direct/RAG/Agent 三按钮。
- JavaScript 通过语法检查。
